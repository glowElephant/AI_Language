# AI_Language 프로젝트 체크리스트

## Phase 1: 언어 설계 (Language Specification)

- [x] **1.1 핵심 문법 명세서 작성** (`docs/spec/grammar.md`)
  - 메시지 기본 구조 정의: `SENDER→RECEIVER: PAYLOAD`
  - 구분자 규칙 (`:`, `|`, `.`, `/` 등의 역할)
  - 데이터 타입 표현법 (문자열, 숫자, 불리언, 리스트, 맵)
  - 중첩 구조 표현법
  - 이스케이프 규칙

- [x] **1.2 명령 체계 설계** (`docs/spec/commands.md`)
  - 명령 카테고리 분류 (요청/응답/알림/에러/스트림 등)
  - 공통 동사 사전 정의 (REQ, RES, ERR, ACK, SUB, PUB 등)
  - 도메인 독립적 기본 명령어 세트
  - 명령 축약 규칙 (약어 생성 패턴)

- [x] **1.3 확장 시스템 설계** (`docs/spec/extensions.md`)
  - 도메인별 확장 네임스페이스 등록 방식
  - 커스텀 명령어/약어 정의 프로토콜
  - 스키마 교환 메커니즘 (에이전트 간 확장 어휘 공유)
  - 버전 호환성 규칙

- [x] **1.4 멀티 에이전트 통신 프로토콜** (`docs/spec/protocol.md`)
  - 1:1, 1:N, N:N 통신 패턴
  - 메시지 라우팅 (브로드캐스트, 멀티캐스트, 유니캐스트)
  - 세션/컨텍스트 관리 (`sid`, `ctx` 헤더)
  - 메시지 체이닝 및 참조 (`ref:msg_id`)
  - 에러 핸들링 패턴

- [x] **1.5 실전 예제 모음 작성** (`docs/spec/examples.md`)
  - 기본 요청-응답 시나리오
  - 멀티 에이전트 협업 시나리오 (3개 이상 에이전트)
  - 에러 처리 시나리오
  - 도메인 확장 시나리오 (게임, 웹개발, 데이터분석 등)
  - 자연어 대비 토큰 절감 비교표

## Phase 2: 파서 및 코어 라이브러리 구현

- [x] **2.1 기술 스택 결정 및 프로젝트 구조 설정**
  - 언어/런타임 선택 (범용성, 성능, AI 에이전트 통합 용이성 기준)
  - 프로젝트 디렉토리 구조 생성
  - 의존성 관리 설정
  - 린터/포매터 설정

- [x] **2.2 렉서(Lexer) 구현**
  - 토큰 타입 정의
  - 토크나이저 구현
  - 에러 위치 추적
  - 렉서 단위 테스트

- [x] **2.3 파서(Parser) 구현**
  - AST(Abstract Syntax Tree) 노드 타입 정의
  - 재귀 하강 파서 또는 적합한 파싱 전략 구현
  - 구문 에러 리포팅
  - 파서 단위 테스트

- [x] **2.4 인코더(Encoder) 구현** — 구조화된 데이터 → AI_Language 문자열
  - 메시지 객체를 압축 문자열로 직렬화
  - 자동 축약 적용
  - 인코더 단위 테스트

- [x] **2.5 디코더(Decoder) 구현** — AI_Language 문자열 → 구조화된 데이터
  - 압축 문자열을 파싱하여 메시지 객체로 역직렬화
  - 확장 네임스페이스 해석
  - 디코더 단위 테스트

- [x] **2.6 검증기(Validator) 구현**
  - 문법 유효성 검사
  - 시맨틱 유효성 검사 (알려진 명령어, 타입 체크)
  - 검증 에러 메시지 생성
  - 검증기 단위 테스트
